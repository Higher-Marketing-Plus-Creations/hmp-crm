Workflow Steps

Step by Step Process

1

Create Client

Admin pehle client banayega. Isme client ka naam, company, email, phone aur internal notes save honge.

Base Record
2

Connect Website

Har client ke andar ek ya multiple websites hongi. Har website ko API key, domain whitelist, notification emails aur status milega.

Per Website Setup
3

Attach Forms

Website ke contact, quote, estimate ya kisi bhi lead form ko CRM API se connect kiya jayega. Har form us website ke andar track hoga.

Form Tracking
4

Receive Leads

Lead submit hone par CRM usko save karega, email notification bhejega, aur email sent ya failed status maintain karega.

Delivery Checks
5

Run Website Test

Admin kisi bhi website par Run Test click karega. System live site check, SSL check, load time, issue summary aur uptime history save karega.

Health Snapshot
6

Review Client Workspace

Har client ke workspace me uski sari websites ke liye 10 monitoring cards, form counts, latest success, failed forms aur issues alag se nazar aayenge.

No Mixed Data
Monitoring Features

10 Features on Each Client Workspace

1. Website online/offline

Live availability status.

2. Email delivery status

Sent, warning, failing, pending.

3. Forms submitted this month

Monthly submission count.

4. Last successful form

Last sent/working inquiry.

5. Failed form count

Submission or delivery failures.

6. Site load time

Response time in milliseconds.

7. Current issues

SSL, downtime, email or no submissions.

8. Run Test button

Manual check trigger.

9. SSL status

Valid, expired, expiring soon.

10. Uptime percentage

Saved check history se calculate hoga.

Suggested real flow:

Create Client -> Connect Website -> Connect Form/API -> Lead Save + Email -> Run Test -> Client Workspace Review
